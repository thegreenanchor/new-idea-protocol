# Phase 4: Visual Artifacts

## Wizard-of-Oz prototype

If requested, create a self-contained HTML prototype.

Rules:

- No build system.
- Make fake/manual parts obvious.
- Demonstrate the happy path.
- Do not imply production auth, payments, security, persistence, or integrations exist.
- Treat the prototype as a comprehension check.

## Visual spec

If requested, create a styled one-page HTML overview of the idea from `templates/VISUAL-SPEC.html`.

Rules:

- Generate it only after the four gates are recorded. Ideas that die in the pressure test die as plain text.
- Self-contained HTML, no build system.
- Fill every placeholder from the spec content. Remove optional blocks that do not apply.
- Restyle to your own design system if you have one; the structure matters more than the skin.
- The Markdown spec remains the source of truth. On any conflict, the Markdown wins.
