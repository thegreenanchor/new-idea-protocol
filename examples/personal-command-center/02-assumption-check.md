# Clarify + Pressure Test

## Riskiest user assumption

Users will trust a generated brief from a messy transcript if the system shows assumptions and unanswered questions.

## Riskiest technical assumption

Transcript quality will be good enough to extract projects and next actions without a long correction loop.

## What can be faked manually

Voice transcription can be manual in v1. Paste transcript text into the tool instead of building recording and speech-to-text first.

## What not to build in v1

- Mobile app.
- Team accounts.
- Calendar integration.
- Automatic task creation.
- Real-time recording.
